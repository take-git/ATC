# C問題について
ナイーブな実装をしたのになぜか通ってしまった。が計算量を減らすための工夫がある。
詳しくはリンクを見ると良いが、与えられたNに応じてひたすら交換を行うのではなく、ループが怒るタイミングを見つけることで計算量を削減できる。
この場合、30回処理を行うと元の1 2 3 4 5 6 と言う並びに戻る。つまりN　mod　30を利用して30通りだけみてあげたらいい。
